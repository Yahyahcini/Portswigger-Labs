## 🧪 Lab Structure

Each lab write-up follows a consistent format:

- **Vulnerability Type** – What category of flaw is being exploited  
- **Exploitation Steps** – Exact process used to identify and exploit the issue  
- **Payload(s) Used** – Precise inputs that triggered the vulnerability  
- **Key Takeaway** – Core concept or pattern learned  
- **Screenshot Proof** – Evidence of successful exploitation  

---

## 🎯 Objective

The purpose of this repository is to develop practical web security skills through repetition and pattern recognition.  

No filler content. Only direct exploitation techniques and real payloads.
